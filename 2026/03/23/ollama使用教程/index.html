Ollama 简介

Ollama 是一个开源的本地大语言模型（LLM）运行框架，允许用户在本地计算机上运行和部署各种开源 AI 模型。它提供了一个简单易用的 API 接口，支持模型管理、文本生成、对话聊天、向量嵌入等功能。

主要特性：

  • 本地运行，无需联网
  • 支持多种开源模型（Llama、Mistral、Qwen 等）
  • 提供 RESTful API 接口
  • 支持流式响应
  • 提供 OpenAI 兼容 API
  • 模型管理（下载、删除、复制等）

默认地址： http://localhost:11434

Ollama 提供两类 API 端点：

  • 原生 API：/api/*（推荐使用）
  • OpenAI 兼容 API：/v1/*

1. /api/generate - 文本生成

HTTP 方法： POST

请求参数：

参数名 类型 必填 默认值 描述
model string 是 - 模型名称（如 “llama3.2”）
prompt string 是 - 生成文本的提示词
stream boolean 否 false 是否使用流式响应
system string 否 - 系统提示词（角色定义）
template string 否 - 消息模板
context array 否 - 上下文数组（来自之前的响应）
options object 否 - 模型参数选项
images array 否 - 图像 base64 字符串（视觉模型）
keep_alive string 否 5m 模型在内存中保持时间

options 对象参数：

参数名 类型 默认值 描述
temperature float 0.8 温度参数（0.0-2.0），控制随机性

num_ctx int 2048 上下文窗口大小
num_predict int 128 最大生成 token 数量
stop array [] 停止序列数组
seed int random 随机种子
repeat_penalty float 1.1 重复 token 惩罚
presence_penalty float 0.0 已存在 token 惩罚
frequency_penalty float 0.0 token 频率惩罚

最佳实践

  1. 使用流式响应：对于长文本生成，使用流式响应可以提供更好的用户体验
  2. 合理设置参数：
    • temperature: 创造性任务设为 0.7-0.9，确定性任务设为 0.1-0.3
    • num_predict: 根据需求设置，避免过长响应
    • top_p: 通常设为 0.9-0.95
  3. 错误处理：始终检查 HTTP 状态码和错误响应
  4. 超时设置：为长时间运行的操作设置适当的超时
  5. 上下文管理：对于长对话，注意上下文长度限制
  6. 模型选择：根据任务选择合适的模型：
    • 通用对话：llama3.2, qwen3
    • 代码生成：codellama, deepseek-coder
    • 嵌入向量：nomic-embed-text, bge-m3

常见问题

Q: 如何更改 Ollama 默认端口？
A: 设置环境变量 OLLAMA_HOST=0.0.0.0:11435

Q: 如何查看 API 请求日志？
A: 启动 Ollama 时添加 --verbose 参数

Q: 如何加速模型加载？
A: 使用 keep_alive 参数保持模型在内存中

Q: 如何处理长文本？
A: 使用流式响应并分块处理，或使用上下文窗口扩展技术
