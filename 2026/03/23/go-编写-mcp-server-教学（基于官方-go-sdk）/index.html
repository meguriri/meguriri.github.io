Go 编写 MCP Server 教学（基于官方 go-sdk）

1. MCP 是什么

MCP（Model Context Protocol）可以理解为让大模型通过标准协议调用你程序能力的一套规范。

你写一个 MCP Server，本质就是：

  1. 声明自己是谁（服务名、版本）
  2. 注册可调用能力（Tool）
  3. 在 Tool 被调用时返回结构化结果
  4. 通过某种传输方式对外提供服务（最常见是 stdio）

3. 写一个 MCP Server 需要哪些核心组件

3.1 Server 实例

作用：承载所有功能（tools/resources/prompts）。

核心函数：

  • mcp.NewServer(impl *mcp.Implementation, options *mcp.ServerOptions) *mcp.Server

关键参数：

  • impl.Name
    • 服务名，客户端会看到
  • impl.Version
    • 服务版本，方便维护和排错
  • options
    • 可选高级配置（中间件、初始化钩子等），最小实现可传 nil

3.2 Tool 定义

作用：告诉客户端你提供了什么能力。

核心函数：

  • mcp.AddTool(server, tool, handler)

关键参数：

  • server
    • 由 NewServer 创建出来的服务对象
  • tool.Name
    • 工具唯一标识，客户端通过它调用
  • tool.Description
    • 给模型看的说明，越清楚越容易被正确调用
  • handler
    • 工具执行逻辑

3.3 Tool Handler（处理函数）

作用：真正处理请求并返回结果。

常见签名：

  • func(ctx context.Context, req *mcp.CallToolRequest, args T) (*mcp.CallToolResult, any, error)

参数意义：

  • ctx
    • 请求上下文，可用于超时、取消、链路控制
  • req
    • 原始调用请求，可拿到调用级信息
  • args
    • 解析后的参数对象（由 SDK 根据 JSON 入参映射）

返回值意义：

  • *mcp.CallToolResult
    • 返回给客户端的结果主体，通常放 Content
  • any
    • 结构化返回（可选），最小实现可用 nil
  • error
    • Go 层错误；最小实现一般返回 nil

3.4 传输层（Transport）

作用：让客户端与服务通信。

最小实现常用：

  • &mcp.StdioTransport{}

配合函数：

  • server.Run(context.Background(), &mcp.StdioTransport{})

含义：

  • 使用当前进程的标准输入输出作为通信通道
  • 非常适合 Claude Code / Claude Desktop 这类通过子进程启动 MCP Server 的场景
